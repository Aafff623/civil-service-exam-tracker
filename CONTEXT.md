# CONTEXT.md — 公务员考试学习跟踪系统

## Project domain

这是一个帮助大学生备考公务员考试的**学习跟踪系统**。系统围绕"用户 - 目标 - 计划 - 练习 - 反馈"的闭环设计，帮助考生制定计划、记录进度、识别弱项、获取推荐。

## Core concepts

### User（用户）
使用系统的考生。每个用户有独立的备考目标、学习计划、答题记录和进度数据。

### Goal（学习目标）
用户设定的备考参数，包括：
- 目标考试类型（如国考、省考）
- 计划开始日期
- 计划考试日期
- 每日可用学习时长（分钟）

系统根据目标计算备考周期，并生成学习计划。

### Subject（考试科目）
公务员考试包含的科目或知识点分类，例如：
- 行政职业能力测验（行测）
  - 言语理解与表达
  - 数量关系
  - 判断推理
  - 资料分析
  - 常识判断
- 申论

每个科目有重要性权重和难度系数，用于计划生成。

### Resource（学习资源）
备考资料，包括：
- 考试大纲
- 历年真题
- 模拟题
- 学习资料/笔记

资源按科目和类型分类，供用户浏览和学习。

### Question（题目）
题库中的单个题目。包含：
- 题干
- 选项（单选/多选/判断）
- 正确答案
- 所属科目/知识点
- 解析
- 答题技巧

### Answer（答题记录）
用户回答某道题目的记录。包含：
- 用户 ID
- 题目 ID
- 用户选择的答案
- 是否正确
- 答题时间

答题记录是弱项识别和进度统计的数据基础。

### Plan（学习计划）
系统根据用户目标和初始测试结果生成的整体学习计划。一个计划覆盖整个备考周期。

### PlanItem（每日计划项）
学习计划中的每一天任务。包含：
- 日期
- 科目
- 建议学习内容
- 建议学习时长
- 是否完成

### Progress（学习进度）
用户每日学习活动的记录，包括：
- 完成的学习任务
- 学习时长
- 答题数量
- 完成状态

### WeakPoint（弱项）
通过分析答题记录识别出的薄弱科目或知识点。判定规则：
- 某科目正确率低于阈值（默认 60%）
- 答题数量达到一定样本量后才会计入弱项

### Recommendation（推荐）
系统基于用户弱项和学习目标推荐的内容，包括：
- 推荐学习资源
- 推荐同类练习题

### Comment（答疑留言）
用户对某道题目提交的疑问，以及对应的回复。

## Naming conventions

### Database
- 表名：小写，复数形式，下划线分隔，如 `users`、`plan_items`
- 字段名：小写，下划线分隔，如 `user_id`、`created_at`
- 主键：`id`（自增整数）
- 外键：`<表名>_id`，如 `user_id`、`subject_id`
- 时间戳：`created_at`、`updated_at`

### API
- 路径：`/api/<resource>`，复数形式
- 统一响应格式：
  ```json
  {
    "success": true,
    "data": {},
    "message": ""
  }
  ```

### Frontend
- HTML 文件：与页面名一致，如 `login.html`、`dashboard.html`
- CSS/JS 文件：按页面或功能分组

## Business rules

1. 用户必须先登录才能查看个人学习计划、答题记录和进度。
2. 一个用户同时只有一个有效的学习计划；重新生成会覆盖旧计划。
3. 弱项识别只在答题样本量 ≥ 5 题时生效。
4. 推荐内容优先针对正确率最低的科目。
5. 每日计划项默认未完成，用户手动标记完成后更新进度。

## Out of scope

- 真实考试报名
- 支付/会员系统
- 多设备实时同步
- AI 模型驱动的智能推荐（规则推荐即可）
- 移动端原生 App
