# PRD - 公务员考试学习跟踪系统

## 1. Summary

本文档定义一个面向大学生的**公务员考试学习跟踪系统**。系统采用前后端分离架构，后端使用 Python Flask + SQLite，前端为独立演示网站。核心目标是帮助考生管理备考资源、生成每日学习计划、跟踪学习进度，并通过做题历史识别弱项，提供个性化推荐和题目解析。

## 3. Background

### Context
公务员考试内容多、周期长，考生在自学过程中容易缺乏计划、抓不住重点、不知道如何分配时间。很多备考者依靠纸质资料或零散网课，难以系统跟踪自己的学习进度和薄弱环节。

### Why now?
随着在线学习工具的普及，学生期望有一个轻量化的系统来辅助备考。本系统作为课程设计项目，目标是在可控复杂度内，实现一个可用的学习跟踪工具。

### 可行性
当前技术栈成熟，Flask + SQLite 足够支撑中小型课程项目；前端页面可以用纯 HTML/CSS/JS 实现，无需引入重型框架，降低开发和维护成本。

## 4. Objective

### 目标
为备考公务员考试的学生提供一个简单、可用的学习跟踪平台，帮助他们制定计划、记录进度、发现弱项并获取针对性学习资源。

### 关键结果（SMART）
- **功能覆盖**：在 4 周内完成 7 个核心功能模块的开发与联调。
- **用户体验**：单个学习任务从打开系统到开始学习不超过 3 次点击。
- **数据完整性**：用户答题记录、学习进度、计划生成逻辑 100% 持久化到数据库。
- **稳定性**：主要功能在本地演示环境下无明显报错，页面加载时间不超过 3 秒。

## 5. Market Segment(s)

### 目标用户
- **在校大学生**：大一、大二学生，刚开始接触公务员考试，需要系统了解考试内容和制定学习计划。
- **自学备考者**：没有报班、希望低成本自我管理学习的人群。

### 约束
- 项目定位为课程设计，不追求商业化功能（如支付、直播、社交）。
- 用户数据量小，单用户本地使用为主。
- 智能推荐以规则为主，不引入机器学习模型。

## 6. Value Proposition(s)

### 用户获得的价值
- **有计划**：根据可用时间和目标自动生成每日学习任务，减少决策负担。
- **有重点**：通过科目权重和难易度分配时间，帮助用户把精力放在关键内容上。
- **有反馈**：记录做题历史，自动识别薄弱知识点，避免重复犯错。
- **有资源**：集中管理考试大纲、历年试题和解析，减少资料散落。

### 避免的痛苦
- 不知道每天该学什么
- 不清楚自己哪些科目比较弱
- 学习进度靠感觉，难以量化
- 找资料费时

## 7. Solution

### 7.1 UX / 用户流程

#### 核心用户旅程
1. **注册/登录** → 进入个人主页
2. **设置学习目标** → 选择目标考试类型、计划开始时间、每天可用学习时长
3. **初始水平测试** → 做少量题目，系统评估各科基础
4. **查看学习计划** → 系统生成每日学习任务卡片
5. **开始学习** → 查看资源、做题、记录完成状态
6. **查看进度与弱项** → 在统计页面查看完成率和推荐内容
7. **查看解析/留言答疑** → 针对错题查看解析，或提交疑问

#### 主要页面
- 登录/注册页
- 个人主页（Dashboard）
- 学习目标设置页
- 初始测试页
- 学习计划页
- 资源库页
- 题库练习页
- 答题结果页
- 进度统计页
- 个性化推荐页
- 题目解析/答疑页

### 7.2 关键功能

#### 模块 1：用户与账户
- 用户注册（用户名、密码）
- 用户登录/登出
- 个人信息查看与编辑
- 学习目标设置（考试类型、备考周期、每日可用时长）

#### 模块 2：考试资源管理
- 资源分类：考试大纲、历年真题、模拟题、学习资料
- 资源列表展示（标题、类型、科目、上传时间）
- 资源详情查看
- 管理员/后台可新增资源（演示阶段可预置数据）

#### 模块 3：智能学习计划生成
- 根据用户设置的可用时间和备考周期，计算总学习天数
- 根据科目重要性和难易程度分配时间权重
- 结合初始测试成绩，为薄弱科目分配更多时间
- 生成每日学习任务（科目 + 内容 + 建议时长）
- 支持重新生成计划

#### 模块 4：题库与练习
- 按科目/知识点浏览题目
- 单题答题（单选/多选/判断）
- 提交答案并记录正确性
- 保存答题历史
- 自动统计各科目正确率，识别弱项

#### 模块 5：学习进度跟踪
- 记录每日任务完成情况
- 统计已完成天数、总学习时长、答题数量
- 可视化展示（进度条、饼图/柱状图）
- 生成学习周报/月报（简单汇总）

#### 模块 6：个性化推荐
- 根据弱项科目推荐相关学习资源
- 根据错题知识点推荐同类练习题
- 推荐逻辑基于规则（如：正确率 < 60% 的科目优先推荐）

#### 模块 7：题目解析与答疑
- 每道题提供详细解析和答题技巧
- 用户可对题目提交疑问
- 管理员/系统可回复（演示阶段可预置常见答疑）

### 7.3 Technology

#### 架构
- **前端**：HTML5 + CSS3 + JavaScript（原生，无框架）
- **后端**：Python Flask
- **数据库**：SQLite
- **通信**：RESTful API，JSON 格式
- **部署**：本地运行，Flask 开发服务器 + 浏览器访问

#### 数据库核心表
- `users`：用户信息
- `goals`：学习目标
- `subjects`：考试科目
- `resources`：学习资源
- `questions`：题库
- `answers`：答题记录
- `plans`：学习计划
- `plan_items`：每日计划项
- `progress`：学习进度记录
- `weak_points`：弱项统计
- `recommendations`：推荐记录
- `comments`：答疑留言

#### API 设计原则
- 接口路径清晰，如 `/api/users`、`/api/plans`、`/api/questions`
- 返回统一 JSON 结构：`{ "success": true/false, "data": {}, "message": "" }`
- 使用 JWT 或 Session 进行简单登录态管理（课程项目推荐 Session）

### 7.4 Assumptions

- 用户主要在本地单设备使用，暂不考虑多设备同步。
- 智能推荐不依赖 AI 模型，基于规则实现。
- 题库和解析内容在演示阶段可由开发者预置，不开放用户自主录入。
- 系统不处理真实考试报名或支付功能。
- 用户每天实际学习时长可能和计划不一致，系统以用户标记完成状态为准。
