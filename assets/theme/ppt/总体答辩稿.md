# 总体答辩稿

## 开场

各位老师好。我们做的项目是"公务员考试学习跟踪系统"。下面我从三个角度来介绍：为什么要做这个系统、系统能干什么、有什么亮点。

## 为什么要做

公务员考试备考有几个比较普遍的问题。第一，内容太多了，行测加申论，知识点又多又杂，自学的同学经常不知道从哪下手。第二，复习周期很长，半年到一年都很正常，这么长的时间没有计划的话很容易三天打鱼两天晒网。第三，学完之后不知道自己学得怎么样，哪里不行，缺乏反馈。

所以我们就想做一个工具，帮备考的同学做三件事：帮你定计划、帮你记进度、帮你找弱项。简单说就是"备考管家"的角色。

## 系统能干什么

系统的工作流程是这样的：

第一步，用户注册登录之后，先设定自己的学习目标。考什么考试、从哪天开始、到哪天考试、每天能学多久。

第二步，系统根据这个目标，自动生成一份从开始到考试每一天的学习计划。每天学什么科目、做什么练习、学多长时间，都安排好了。

第三步，用户可以按计划学习，在题库里做题。题库有 200 道题，单选、多选、判断都有。做完立刻出结果，还能看解析。

第四步，系统会自动统计你的学习数据。总学习时长、完成率、正确率，都用图表展示出来。还能自动识别你哪些科目比较薄弱。

第五步，针对薄弱科目，系统会推荐相关的学习资料和练习题，帮助你有针对性地补强。

最后，每道题都有解析，不懂的还可以留言提问。

整个流程串起来，用户只需要跟着系统走，不需要自己想"今天该干嘛"。

## 创新点

我们觉得系统有几个比较突出的地方。

第一个是 Mock 模式。这个功能专门为答辩准备的。正常情况需要前端和后端都跑起来才能用。但如果答辩现场后端出了问题，只需要在网址后面加一个参数，前端就能独立运行，所有功能都能演示。相当于给答辩多上了一份保险。

第二个是智能计划生成。不是简单的"每天换一门课"那种轮询。系统会综合考虑科目的重要性、难度，以及用户自己哪些科目比较弱，来给每个科目分配时间。薄弱科目会多安排，重要的科目也会多安排。整个备考过程还分了三个阶段：基础期、强化期、冲刺期，每个阶段的侧重点不一样。

第三个是弱项驱动的推荐。用户做题之后，系统会自动发现他哪个科目不行，然后精准推送相关的资料和题目。推荐理由也很清楚，不会莫名其妙给你推东西。

第四个是整体设计。我们没有做成那种花里胡哨的风格，而是选了比较克制、专业的配色和排版。看起来像个正经的教育工具，不像随便搭的模板。

第五个是部署很方便。换一台电脑演示，只需要装 Python 依赖，跑初始化脚本，启动服务，三步搞定。如果这三步也不想做，Mock 模式也能演示。

## 技术选型

简单说一下为什么选这些技术。

后端选 Flask 而不是 Django，因为 Flask 更轻量，我们这个项目规模用 Flask 刚刚好。

数据库选 MySQL，这是课程要求的一部分，需要用到关系型数据库。

前端没有用 Vue 或 React，选了原生技术，因为答辩时不需要装一堆工具，打开浏览器就能跑。

推荐功能没有用机器学习，因为数据量不够大，用规则实现反而更可靠、更容易解释。

## 总结

总结一下。这个系统做的事情就是：帮备考公务员的同学把零散的自学，变成有计划、有反馈、有重点的系统性备考。

七个核心功能模块全部实现，200 道题库，23 条学习资源，完整的演示数据。前后端分离，Mock 模式兜底，换台电脑就能演示。

以上就是我们项目的整体介绍，谢谢各位老师。
