# 前端答辩稿

## 开场

各位老师好，我来介绍一下我们系统前端的设计思路。

我们做的是一个公务员考试的学习跟踪系统。前端有 10 个页面，从登录注册，到资源浏览、题库练习、学习统计、个性化推荐，整条学习流程全都有对应的页面。

我们没有用 Vue 或者 React 这些前端框架，用的是最基础的原生技术：HTML、CSS、JavaScript。为什么这么选？因为答辩现场的环境不好说，有些电脑装了 Node，有些没有。用原生技术的话，只要电脑上有浏览器，双击就能打开，一行命令都不用敲。这对课程设计来说是最稳的选择。

## 设计风格

整个页面的配色走的是政务风格。主色调是深藏青，强调色是单一蓝色，背景是浅灰白。没有花里胡哨的渐变，也没有 emoji 当图标。标题和正文分别用了两套经过验证的 Google 字体，读起来很舒服。

我们特意避免了现在很多网站上常见的一些问题：比如按钮颜色太跳、卡片阴影太重、动画晃眼。整个系统看上去是克制的、专业的，符合教育类产品的气质。

## 主要页面

挑几个核心页面说一下。

登录页：简洁的表单，输完用户名密码直接登录，出错有提示。我们提供了四个演示账号，评委想试哪个都行。

资源库：展示了 23 条备考资料，可以按科目和类型筛选。管理员登录后还能上传新资源或者批量删除。

题库练习：这是用得最多的页面。200 道题，分成单选、多选、判断三种。点选项提交，立刻告诉你对错，还能看到详细解析。做完的题会记录在后台，用来算你的正确率。

学习统计：用图表展示你的学习情况。学了多长时间，完成了多少任务，正确率是上升还是下降，一目了然。还有一个打卡日历，哪天学了、哪天没学，清清楚楚。

学习推荐：系统会根据你哪个科目答得不好，自动推荐相关的资料和题目。不用自己去找，系统帮你挑。

## Mock 模式

这个功能专门为答辩设计的。

正常使用需要同时启动前端和后端两个服务。但万一答辩现场后端出了问题（比如 MySQL 连不上、端口被占），我们有一个备选方案：在网址后面加一个参数 `?mock=1`，前端就能脱离后端独立运行。所有数据都是预置好的，页面功能一个不少。

这个模式让我们在答辩的时候多了一重保障。

## 交互体验

我们在细节上做了一些让操作更顺手的设计。比如点击导航栏切换页面的时候有过渡动画，不会白屏刷新。统计页面的数字是从零滚动到目标值的，看起来比较直观。手机端也能正常显示，侧边栏会自动收起来。

这些细节单独看都不大，但加起来会让整个系统用起来更顺畅。
