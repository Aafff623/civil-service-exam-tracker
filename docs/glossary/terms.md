# 术语速查(Glossary)

项目核心术语的中英对照与一句话定义。**详细业务规则见根 [`CONTEXT.md`](../../CONTEXT.md)**;本文件只做速查。

## 领域实体

| 中文 | English | 一句话 |
|---|---|---|
| 用户 | User | 使用系统的考生;有独立目标、计划、答题记录、进度 |
| 学习目标 | Goal | 备考参数:考试类型 + 起止日期 + 每日可用时长 |
| 考试科目 | Subject | 公务员考试科目分类(行测 / 申论及其子模块),共 7 个种子 |
| 备考资源 | Resource | 关联到科目的学习资料(公告 / 资料 / 真题 / 模拟题),共 23 条种子 |
| 题目 | Question | 题库练习单元,单选 / 多选 / 判断,共 200 题种子 |
| 答题记录 | Answer | 用户提交的答案 + 判分结果 + 关联题目 |
| 学习计划 | Plan | 按 Goal 生成的每日任务集合;一个用户同时只有一个有效计划,重新生成会覆盖 |
| 计划项 | Plan item | 每日任务,可逐项标记完成 |
| 学习进度 | Progress | 完成率、答题量、正确率等统计 |
| 弱项 | Weak point | 答题样本 ≥ 5 且正确率 < 60% 的科目 |
| 推荐 | Recommendation | 针对最弱科目推荐资源 + 同类题(规则驱动,非 ML) |
| 答疑留言 | Comment | 题目下的留言,规则化自动回复 |

## 角色

| 中文 | English | 一句话 |
|---|---|---|
| 管理员 | admin | 可增删资源 / 科目;演示账号 `root` |
| 普通用户 | user | 仅操作自己的数据;演示账号 `testuser1 / 2 / 3` |

## 技术术语

| 中文 | English | 一句话 |
|---|---|---|
| Mock 模式 | Mock mode | 无后端时前端用 JS 层 mock 数据演示;URL `?mock=1` 或 `localStorage.MOCK_MODE` 开启 |
| 种子数据 | Seed | `db/seed/init_db.sql` 的演示数据,由 `init_db.py` 删库重建后导入 |
| 资源关联 | resource_id | 题目通过 `resource_id` 关联资源,可从资源详情进入关联题目练习 |
