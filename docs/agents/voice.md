# Voice

> **Output style**: `humanizer-output-style` skill — 统一语气、去 AI 味。详见全局 `skills/humanizer-output-style/SKILL.md`。

所有文档、commit message、handoff、agent 输出均遵循以下语气。

## 项目语气

- **语言**：中文为主（项目是中文课设，面向大学大一、大二学生）；代码标识符、API 路径、技术术语保留英文。
- **口吻**：平实、直接、口语化。避免「赋能」「闭环」「抓手」等空话；避免「首先……其次……最后」的机械连接。
- **面向读者**：答辩老师 + 同学 + 接手的下一个 agent。讲清「做了什么、为什么、怎么复现」，不讲产品手册式的空话。
- **事实纪律**：数字必须可核实（git log / `db/seed/` / 源码 / 行号）。不编造、不四舍五入到好看的数。

## 去 AI 味清单

- 不堆 emoji 装饰（功能性图标除外，如表格里的 ✅/⚠️）。
- 不用「无疑」「显而易见」「值得注意的是」等填充词。
- 不写过度对称的排比和三段式总结。
- 不在结尾加「综上所述」「相信通过……」式收束。
- 列表只在该用列表时用，不为排版而拆。

## 应用范围

`docs/*.md`、`README.md`、`CLAUDE.md`、`AGENTS.md`、commit message、handoff、agent 回复。代码注释保持简洁，匹配周围代码风格，不单独发挥。
